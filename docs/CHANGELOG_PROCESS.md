# Changelog & Commits

## Commit Guidelines
Adopt Conventional Commits standards for all code modifications:
- `feat`: A new feature addition.
- `fix`: A bug resolution.
- `docs`: Documentation updates.
- `refactor`: Restructuring code with no feature changes.
- `perf`: Optimizations for performance.

## Release Process
- Tag versions using semver (e.g. `v1.2.0`).
- Maintain a running `CHANGELOG.md` listing additions and improvements.
